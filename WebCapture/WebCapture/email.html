FAQ

  • Which builds qualify as dogfood builds?

(N-1) public Preview or newer builds qualify as valid dogfood builds. For e.g. Current latest public Preview is 17.0 Preview 4.x. Hence, any build newer than or equal to the last build released for 17.0 Preview 3 cycle is a valid dogfooding build.

  • Which builds qualify to get credit for “On the latest build” metric?

Latest (N) public Preview or newer internal builds qualify as valid “latest” build. Any usage on these will get credit for the “On the latest build” metric. For. e.g. Currently threshold for “Latest Build” is 17.0 Preview 4.1.

  • What does “Primarily using builds from this channel” mean?

In cases where a user has used multiple versions of VS in last 7 days, the version that has the most usage as seen in telemetry is considered as the “primarily used version”. This is indicated by the purple star above. This is a rolling 7 day metric. For e.g. If the user has used a VS 2022 build that meets the criteria to qualify as dogfood build, then WhoDogfoods will show a purple star in “Dev17 Dogfooding” column for that user.

Note: If the user has just started using VS 2022 dogfood build, it might take some usage from the user on the dogfood builds to change the Primary IDE indication on the site from the older version that the user was using.

  • How frequently should one update the build?

Nightly internal builds – please update at least once a week.

Public Preview builds – please update anytime a new build is released to this channel.

  • Are employees who do not use VS at all expected to dogfood?

No. The goal is not to force everyone to use Visual Studio but rather, encourage those who are using older versions of Visual Studio to use the latest VS 2022 builds.

  • Where can I find information for Visual Studio Preview builds?

Go to ShipReady Releases Page. Set “Any major release” filter to 17 to see publicly released Preview builds information.

Please email WhoDogfoodsReports and specify the reason.
